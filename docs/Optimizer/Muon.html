Singular Value Decomposition

如果一个矩阵 A∈Rm×n, 那么它可以分解为：

A=UΣV⊤

这三个不同的矩阵，其中：

  • U: Left Singular Vectors, 输出空间的正交方向
  • Σ： Singular Values Matrix，表示在每个方向上的拉伸强度
  • V: Right Singular Vectors，输入空间里的正交方向

我们也可以把它看成，一个矩阵经过：旋转+拉伸+旋转

我们知道， 在 AdamW 更新的时候，有一个Momentum，研究表明，这个Momentum Matrix是Low Rank Matrix，这就意味着，只有少部分的方向parameters对更新做出了贡献，这在深度学习中是in-efficient的。

Low Rank Matrix

矩阵里真正独立的信息很少，虽然表面上它很大，但本质上只由少数几个方向决定。比如一个 1000×1000 的矩阵：

  • 如果 rank = 1000，说明信息很丰富，很多方向都独立
  • 如果 rank = 3，说明虽然它有一百万个元素，但本质上只需要 3 个独立方向就能描述大部分结构

Ortho(M)s.t.OO⊤​=Oargmin​{∥O−M∥F​}=I or O⊤O=I​

但是这个 O 在High-Dimensional的空间中，是很难找到的，因此用到了SVD这个工具。

通过把 Σ 设置成Singular Value， 我们就得到了:

M=UΣV⊤≈UV⊤

这个 UV⊤ 就是Orthogonal Matrix：

(UV⊤)(VU⊤)=U(V⊤V)U⊤=UIU⊤=I

Moun 是一个用于2D parameters的优化器，a.k.a 用于Linear Layer。它的创新点在于:

Muon (MomentUm Orthogonalized by Newton-Schulz) optimizes 2D neural network parameters by taking the updates generated by SGD-momentum, and then applying a Newton-Schulz (NS) iteration as a post-processing step to each of them before applying them to the parameters.

首先我们来看一下 Bt​ 是什么。Bt​ 是一个缓冲向量，也就是SGD，Adam里面的Momentum。

Newton-Schulz

Moun的创新点就在与NewtonSchulz， NewtonSchulz 是一种矩阵迭代方法， 它的作用是：不用直接做很贵的矩阵分解，而是通过反复迭代，去近似计算某些矩阵函数。如果一个矩阵：

M=UΣV⊤

Muon 理想上想把它变成：

M≈UV⊤
  • 保留方向结构 U, V
  • 去掉奇异值 Σ 的尺度不平衡

但是问题是直接计算SVD很复杂，这时候我们就可以使用 NewtonSchulz，它的价值在于：

它可以不用显式做 SVD，只靠几次矩阵乘法，就近似得到这个正交部分。

当然，向之前的提到的，Moun 只适用于2D的Parameters，因此，当使用Moun优化模型时，需要与 AdamW 等优化器结合起来使用：

  • Moun 负责2D的Parameters，Linear Layer
  • AdamW 负责 Scalar value或者 Vector value

Muon in LLM

有人在LLM中做了研究，返现如果用纯的Muon，那么随着Training step的增加，它与AdamW的差距就越来越小了，因此这篇文章，重新更新了Muon的更新规则，主要是增加了：

  • Weight Decay
  • Learning Rate Adjust

因此，更新的规则变成了：

θt​←θt−1​−η(0.2 ⋅ max(n,m)​Ot​ +λθt−1​)

Kimi

Kimi的团队在用Muon训练时，发现了另一个问题，就是Attention Logits会随着训练的时间增大，因此它们提出了QK-CLIP的概念， 回顾一下Attention的计算：

S​=QK⊤=(XWQ)(XWK)⊤=(XWQ)(WK⊤X⊤)=X(WQWK⊤)X⊤​

我们知道，X 在传入Attention之前通常是会Normalize的（Pre-Norm），所以问题只能出在WQWK⊤ 上，因此，我们只需要设定一个特殊的Thershold τ， 如果 Smax​>τ, 并且重新Scale the S

S=X(γ​WQγ​WK⊤)X⊤

Why Muon is Fast?

可以把它想象成很多人推一个箱子：

  • 普通的优化器：只有几个人在用力，其他人几乎没有动
  • Muon：所有人朝着同一个方向一起推
